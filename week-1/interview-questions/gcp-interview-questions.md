# Interview Questions for Google Cloud Platform (GCP)

* What is the difference between a region and a zone in Google Cloud?
* Why would you deploy resources across multiple zones within the same region?
* What is the difference between zonal, regional, and global resources in GCP?
* What is Compute Engine?
* What types of workloads are best suited for Compute Engine?
* What is the difference between predefined and custom machine types in Compute Engine?
* What is Cloud Storage, and how is it different from Persistent Disk?
* What is a bucket in Cloud Storage, and what kinds of data are typically stored there?
* What is a Managed Instance Group (MIG), and how does it differ from an unmanaged instance group?
* What are the benefits of using a Managed Instance Group?
* What is autoscaling in a MIG, and when would you use it?
* What is autohealing in a Managed Instance Group?
* How do VPC firewall rules work in Google Cloud?
* What is the difference between ingress and egress firewall rules?
* What is a VM image in Compute Engine, and how is it used?
* What is block storage in Google Cloud, and when should you use it?
* What is the difference between Local SSD and Persistent Disk?
* What are the tradeoffs between temporary and durable block storage?
* What is Cloud SQL, and what database engines does it support?
* What operational tasks does Cloud SQL manage for you?
* What is the difference between a Cloud SQL instance with a public IP vs. private IP?
* How does Cloud SQL provide high availability?
* When would you choose Cloud SQL over running a database directly on Compute Engine?
* How does Persistent Disk provide durability compared to local storage attached to a VM?
* Why might you choose Cloud Storage over block storage for an application?
* How do Google Cloud projects relate to service resources like VMs, buckets, and databases?
